# Adaptive Local Request Owner-Decision Research

Status: ACTIVE PLAN

## Goal

Resolve the architecture's open owner decisions pragmatically from current source contracts, retained LabKit evidence, and primary external documentation. Produce recommended defaults that can be approved now, configurable provisional policies that require shadow calibration, and explicit blockers that cannot be closed without live evidence.

## Boundaries

```text
PRIMARY WORKDIR: repository root
BOARD: lmstudio-labkit
HOST APPLICATION: read-only source/tests/docs
MAY EDIT: this plan; unique research report pairs; final owner-decision brief
MUST NOT EDIT: host source/config/prompts/database; LabKit runtime code/tests/configs/registries; prior reports
NO LIVE ACTIONS: no model load/download/inference, exact-token recapture, cloud calls, benchmarks, migration, commit or push
```

## Decision clusters

1. Summary placeholder, legacy prompt disposition, and microphone-command retention.
2. Approved catalog governance, exact identity/digest, telemetry privacy, and complete JSON Schema validation.
3. Token/output/call defaults, CUDA headroom, Apple unified-memory policy, and multi-device restrictions.
4. Task-specific semantic acceptance and shadow/canary evidence thresholds.
5. Final owner-decision synthesis.

## Research method

- Read current application/LabKit contracts and all 2026-07-13 architecture reports first.
- Prefer official LM Studio, NVIDIA, Apple/platform, and JSON Schema/library documentation for external claims.
- Cite external URLs and clearly label static evidence, retained executed evidence, proposed defaults, and live-unverified assumptions.
- Prefer a small safe policy with typed unavailable outcomes over broad auto-detection.
- Do not invent universal memory/token thresholds when the evidence only supports configurable starting defaults.

## Done conditions

- Four independent Markdown+JSON report pairs exist and JSON parses.
- Final brief assigns each open decision one of: `approve_now`, `approve_configurable_default`, `requires_shadow_calibration`, or `blocked_external_capability`.
- Every recommendation has rationale, failure behavior, rollback seam, and minimum implementation acceptance test.
- No implementation card is dispatched.
